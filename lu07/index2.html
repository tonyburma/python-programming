Flask Web Development

From "Hello World" to Full-Stack Applications

Based on Lecture Topics

1. Introduction to Flask

Transitioning from static logic to dynamic servers.

What is Flask?

  • A micro-framework written in Python.
  • Lightweight and modular.
  • Provides tools, libraries, and technologies to build a web app.

2. Routing & Hello World


                        from flask import Flask
                        app = Flask(__name__)

                        @app.route("/")
                        def index():
                            return "Hello, world!"

Decorators

The @app.route("/") decorator tells Flask which URL should trigger our function.

3. Rendering HTML & Jinja

Separating logic (Python) from presentation (HTML).

Jinja Syntax

  • render_template("index.html") looks in the /templates folder.
  • Placeholders: {{ name }}
  • Control Flow: {% if ... %} and {% for ... %}

4. Handling User Input

GET vs. POST

Method Data Location Best For
GET URL (?name=x) Searching/Fetching
POST HTTP Body Sensitive Data/Changes

5. Template Abstraction

Use layout.html to define a "blueprint" for your site.


                    <body>
                        {% block body %}{% endblock %}
                    </body>

Then {% extends "layout.html" %} in other files.

6. Server-Side Validation

Never trust the user.

  • Users can bypass HTML restrictions using Inspect Element.
  • Always re-validate input in app.py.

7. Data Persistence

Dictionaries vs. SQL

  • RAM: Fast but volatile (data lost on restart).
  • SQL: Permanent storage using db.execute.

Deregistration

Using <input type="hidden"> to pass IDs for deletion without showing them to users.

8. MVC Architecture

  • Model: Your Data (SQL)
  • View: Your UI (HTML/Jinja)
  • Controller: Your Logic (Python/Flask)

9. Cookies & Sessions

HTTP is stateless (it forgets you immediately).

The Solution

  • Cookies: Small files stored on the client.
  • Sessions: Server-side memory tied to a cookie ID.
  • Used for: Logins and Shopping Carts.
