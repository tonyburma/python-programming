Why File Handling?

Data in memory (variables) is volatile—it disappears when the program ends.

  • Persistence: Saving data permanently to the disk.
  • Scalability: Processing data larger than our RAM.
  • Interoperability: Sharing data between different programs and systems.

Data Analytics: Common File Types

  • CSV (Comma Separated Values): The bread and butter of data tables.
  • JSON (JavaScript Object Notation): Standard for web APIs and structured configurations.
  • TXT (Plain Text): Used for logs, raw unstructured data, and documentation.

Understanding File Modes

Modes define how we interact with the file resource.

Mode Description
'r' Read (default)
'w' Write (Overwrites existing)
'a' Append (Adds to end)
'r+' Read & Write

The 'os' Module: Bridging Python & The System

The os library provides a portable way of using operating system-dependent functionality.

  • Navigation: Moving around your computer's folders.
  • File System Manipulation: Creating, deleting, and renaming files/folders.
  • Path Management: Dealing with file locations securely.

1. Navigation & Environment

Where are we? Where are we going?

import os
# Example 1: Get current folder
print(os.getcwd())

# Example 2: List contents of a folder
print(os.listdir('.'))

# Example 3: Change directory
os.chdir('/Users/data_project')

2. File System Manipulation

Creating and cleaning up your workspace.

# Example 1: Create a new directory
os.makedirs('reports/2026')

# Example 2: Rename a file
os.rename('data.txt', 'final_data.txt')

# Example 3: Remove a file
os.remove('temp_file.log')

3. Path Management (The Pro Way)

Hardcoding paths like "C:\Users\..." causes bugs on Mac/Linux. Use os.path instead. 

# Example 1: Join paths safely
path = os.path.join('data', 'raw', 'input.csv')

# Example 2: Check if file exists
if os.path.exists(path):
    print("Ready to process!")

# Example 3: Get absolute path
print(os.path.abspath('data.csv'))

Checking & Opening Files

Before reading, it's best practice to verify existence to prevent crashes. 

import os
# Example 1: Check existence
if os.path.exists("data.csv"): 
    print("File found")

# Example 2: The 'with' context manager
with open("data.csv", "r") as f:
    pass # File is automatically closed

# Example 3: Absolute path opening
with open("/home/user/data.csv", "r") as f:
    pass

Reading Files

# Example 1: Read all at once
data = f.read()

# Example 2: Read line by line
line = f.readline()

# Example 3: Read all lines into a list
lines = f.readlines()

Writing Files

# Example 1: Write a single string
f.write("Hello World")

# Example 2: Write a list of lines
f.writelines(["Line 1\n", "Line 2\n"])

# Example 3: Append data without wiping
with open("log.txt", "a") as f:
    f.write("New Log Entry")

File Exception Handling

Files are external dependencies—they are prone to failure.

# Example 1: Missing file
try:
    f = open("missing.txt", "r")
except FileNotFoundError:
    print("The file does not exist.")

# Example 2: Permission error
except PermissionError:
    print("No access to this file.")

# Example 3: Generic Cleanup
finally:
    print("Attempted file operation.")
