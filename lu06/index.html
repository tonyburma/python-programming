Modern Python Package Management

Transitioning to uv

Based on uv Documentation

Why Virtual Environments?

Global installations (pip install) lead to three major issues:

  • Dependency Conflicts: Different projects needing different versions of the same package.
  • System Pollution: Messy, hard-to-debug global installations.
  • "Works on My Machine": Lack of isolation makes code non-portable.

The Solution

A virtual environment (venv) acts as an isolated "container" for each project.

What is uv?

An extremely fast Python package manager and resolver written in Rust by Astral Software.

  • Replaces pip, venv, pip-tools, and pyenv.
  • 10-100x faster than pip.
  • Unified workflow into one tool.

uv vs. Traditional pip

Feature Traditional pip uv
Speed Slower (Python) Instant (Rust)
Tools Needed pip, venv, pyenv, etc. Just uv (All-in-one)
Lock Files Manual/Incomplete Automatic uv.lock
Activation Manual source venv/bin/activate Automatic via uv run

Analogy: If pip is a screwdriver, uv is a power drill.

Installation

macOS & Linux:

curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell):

powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

Restart your terminal after installation.

Step 1: Initialize Project

uv init my-project
cd my-project

Creates pyproject.toml, .python-version, and hello.py.

Step 2: Add Dependencies

uv add requests
uv add --dev pytest ruff

Updates pyproject.toml and uv.lock automatically.

Step 3: Run Your Code

uv run python hello.py
uv run pytest

Note: You never need to manually activate the venv!

Reproducing Environments

When sharing a project, others can run:

uv sync

This reads uv.lock to create an identical environment.

For one-off scripts:

uv run --with requests python script.py

Summary

  1. Use environments to avoid dependency hell.
  2. uv is faster and simpler than pip + venv.
  3. pyproject.toml is the modern standard.
  4. uv.lock ensures reproducibility across teams.
  5. uv run eliminates manual activation.
