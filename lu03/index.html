LU03: Errors & Exception Handling

Python Programming (CS2001)

Moving from "It crashed" to "I handled it."

What are Exceptions?

An exception is an error that happens during execution. If not handled, the program terminates.

  • Syntax Error: "Grammar" mistakes (e.g., missing colon).
  • Exception: Logical errors (e.g., dividing by zero).

The try-except Block

Used to catch and handle exceptions so the script keeps running.


try:
    # Code that might fail
    result = 10 / 0
except ZeroDivisionError:
    # Code that runs if error occurs
    print("You can't divide by zero!")

Else and Finally

  • else: Runs if NO exceptions were raised.
  • finally: ALWAYS runs, regardless of an error (perfect for closing files).

try:
    f = open("test.txt", "r")
except FileNotFoundError:
    print("File missing")
else:
    print("Read successful")
finally:
    print("Cleaning up resources...")

Using Assertions

Assertions are sanity checks for your code. They test if a condition is True; if not, they raise an AssertionError. 


def calculate_age(year):
    assert year > 0, "Year must be positive!"
    return 2024 - year

Why Custom Exceptions?

Standard exceptions like ValueError are too generic for specific business rules.

  • Clarity: InsufficientFundsError is clearer than a generic RuntimeError.
  • Precision: You can catch your specific error without affecting others.
  • Data: Custom exceptions can carry extra attributes (like a current balance or user ID).

The Anatomy of a Custom Error

All custom exceptions must inherit from the built-in Exception class.


class ValueTooHighError(Exception):
    """Raised when the input value is too high"""
    def __init__(self, value, limit):
        self.value = value
        self.limit = limit
        self.message = f"{value} exceeds limit of {limit}"
        super().__init__(self.message)

Topic 2: Custom Exceptions (3 Examples)

1. Business Logic: MinAgeError for a voting app.

2. Security: InvalidTokenError for an API authentication module.

3. Data Integrity: NegativeStockError for an inventory system.

Example: The Inventory Guard


def update_stock(count):
    if count < 0:
        raise NegativeStockError(count)
    print(f"Stock updated to {count}")

try:
    update_stock(-5)
except NegativeStockError as e:
    print(f"Inventory Error: Cannot set stock to {e.value}")

Creating Custom Exceptions

Sometimes built-in errors aren't specific enough. We can create our own by inheriting from the Exception class. 


class SalaryNotInRangeError(Exception):
    """Exception raised for errors in the input salary."""
    def __init__(self, salary, message="Salary not in (5k, 30k)"):
        self.salary = salary
        self.message = message
        super().__init__(self.message)
