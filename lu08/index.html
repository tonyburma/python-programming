SQLAlchemy with Flask

Database Management with SQLite3

Object-Relational Mapping (ORM) Essentials

What is SQLAlchemy?

  • An Object-Relational Mapper (ORM) for Python.
  • The Problem: Raw SQL strings are messy and error-prone.
  • The Solution: Interact with database tables as Python classes (objects).

Why Use It?

Feature Benefit
Abstraction Write Python, not SQL; swap backends easily.
Security Automatic protection against SQL injection.
Maintainability Schema defined in version-control friendly code.
Productivity Less boilerplate; focus on logic.

Getting Started


pip install flask flask-sqlalchemy

Note: SQLite3 is built into Python; no separate driver needed.

Basic Configuration


app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///shop.db'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False
db = SQLAlchemy(app)

shop.db will be created in your project root.

Defining Models


class Product(db.Model):
    __tablename__ = 'products'
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    price = db.Column(db.Float, nullable=False)
    # Foreign Key
    category_id = db.Column(db.Integer, db.ForeignKey('categories.id'))
  • db.Column: Defines a table column.
  • primary_key: Unique identifier.
  • nullable=False: Required field.

CRUD Operations

Create


prod = Product(name="Laptop", price=999)
db.session.add(prod)
db.session.commit()

Update


prod.price = 899.99
db.session.commit()

Read


all = Product.query.all()
one = Product.query.get(1)

Delete


db.session.delete(prod)
db.session.commit()

Database Migrations

db.create_all() won't update existing tables. Use Flask-Migrate instead.

  1. flask db init: Setup repository.
  2. flask db migrate -m "message": Generate script.
  3. flask db upgrade: Apply changes.

One-to-Many


# In Category Model
products = db.relationship('Product', backref='category', lazy=True)

Allows category.products and product.category access.

Many-to-Many

Requires an Association Table.


product_tags = db.Table('product_tags',
    db.Column('product_id', db.Integer, db.ForeignKey('products.id'), primary_key=True),
    db.Column('tag_id', db.Integer, db.ForeignKey('tags.id'), primary_key=True)
)

Use secondary=product_tags in your relationship.

The N+1 Problem

Standard relationships load data "lazily," causing multiple DB hits.

The Fix: Eager Loading


from sqlalchemy.orm import joinedload

products = Product.query.options(joinedload(Product.category)).all()

Fetches everything in one SQL query.
