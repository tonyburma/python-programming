What is OOP?

A programming paradigm based on the concept of "objects," which can contain data (attributes) and code (methods).

  • Class: The blueprint (e.g., a "Car" design).
  • Object: The instance (e.g., your specific red Tesla).

Topic 1: Classes and Objects


class Student:
    def __init__(self, name, grade):
        self.name = name  # Attribute
        self.grade = grade

    def display_info(self):  # Method
        print(f"Student: {self.name}, Grade: {self.grade}")

s1 = Student("Alice", "A")
s1.display_info()

Basic OOP: 3 Examples

  1. Bank Account: Attributes (balance, owner) and Methods (deposit, withdraw).
  2. Video Game Character: Attributes (health, mana) and Methods (attack, heal).
  3. Smart Home Device: Attributes (status, brightness) and Methods (turn_on, set_level).

Topic 2: Inheritance

Allowing a class to derive attributes and methods from another class.


class Animal:
    def speak(self):
        print("Animal makes a sound")

class Dog(Animal):  # Dog inherits from Animal
    def speak(self):
        print("Woof!")

Polymorphism & Encapsulation

  • Polymorphism: Different classes using the same method name (e.g., .move() for Bird and Fish).
  • Encapsulation: Hiding internal data from the outside (using __private variables).
  • Abstraction: Showing only necessary features to the user.

Advanced OOP: 3 Examples

  1. Payment System (Polymorphism): CreditCard and PayPal classes both have a process_payment() method.
  2. Employee Hierarchy (Inheritance): Manager and Developer inheriting from Employee.
  3. Secure Login (Encapsulation): A User class where the __password attribute cannot be accessed directly.

Topic 2.2: Polymorphism in Action

Polymorphism allows different classes to be treated as the same "type" through a shared method interface.

Example 1: Payment Systems


class CreditCard:
    def process(self, amt):
        return f"Charging ${amt} to Card"

class PayPal:
    def process(self, amt):
        return f"Redirecting to PayPal for ${amt}"

# Both respond to .process()

Example 2: File Exporters

PDFExporter and CSVExporter both have a .save() method, but write data differently.

Example 3: UI Elements

Button and Checkbox both have a .render() method to draw themselves on screen.

Why use Polymorphism?

  • Flexibility: You can add a CryptoPayment class later without changing your main "Checkout" code.
  • Simplicity: No need for complex if-else blocks to check if an object is a Card or PayPal.
  • Maintenance: Keeps your Advanced OOP structures clean and professional.

Topic 3: Magic Methods

Special methods surrounded by double underscores (Dunder).

  • __init__: Initializes an object.
  • __str__: Defines how an object is printed as a string.
  • __len__: Defines behavior for the len() function.
